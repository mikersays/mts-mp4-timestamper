# Product Guidelines

## Application Name
**MTS Converter** - A simple, descriptive name that clearly communicates the tool's purpose.

## Tone & Voice
- **Professional and straightforward** - Clear, minimal, business-like communication
- Avoid jargon unless necessary for technical accuracy
- Use direct, imperative language for instructions
- No unnecessary filler words or overly casual expressions

## Visual Design Principles
- **Clean and minimal** - Neutral colors, simple layouts, focus on functionality
- Prioritize clarity over decoration
- Use standard Windows UI controls for familiarity
- Adequate whitespace to reduce visual clutter
- Consistent spacing and alignment throughout

## Color Palette
- Neutral grays for backgrounds and borders
- White for primary content areas
- Black/dark gray for text
- Accent color for primary actions (Convert button)
- Red for errors, green for success states

## Typography
- System fonts (Segoe UI on Windows) for native appearance
- Clear hierarchy: titles, labels, body text
- Readable font sizes (minimum 9pt for UI elements)

## Messaging Guidelines

- Include progress indication for long operations

### Error Messages
- Short with clear next steps
- Example: "FFmpeg not found. Ensure ffmpeg.exe is in the application folder."
- Avoid technical stack traces in user-facing messages

### Success Messages
- Confirm completion with relevant details
- Example: "Video saved to: C:\Output\video.mp4"

## Documentation Style
- **Concise quick-start guides** - Get users running fast
- Bullet points over paragraphs
- Step-by-step numbered lists for procedures
- Screenshots only when essential
- Troubleshooting section for common issues
